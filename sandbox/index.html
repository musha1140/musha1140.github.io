WHAT IS A "HOP" / CAPSULE?
A capsule is a plaintext string with the envelope #!TYPE:BODY. In the real /kilo/ app, the plaintext only appears after you decrypt a share link with the correct MIDI — the song is the key. The dispatcher then routes the capsule to a handler based on TYPE (and, for exec, the sub-type).

The dispatcher is a capability cage: the cipher proves who sent it, but what the payload is allowed to do is bounded here, not in the crypto. A hop: capsule can only navigate. A hint: can only display. A solver who walks a chain experiences both mechanics — that's the labyrinth.

The richest capsule is #!exec:lab: — a JSON manifest that turns the message into a setlist. Each "track" is a controlled HOP to another encrypted stage. The user isn't reading the message — they're playing it. Destinations are visible before click, navigation is gated by maxHops, host allowlist, and a confirm dialog.

This sandbox lets you author and fire capsules without the encryption layer so you can study the dispatcher directly. Every run produces a verbose receipt (parser trace, dispatch decision, SHA-256, nonce, mode). SAFE mode blocks navigation, popups, banners, console effects, and script behavior; lab manifests may render as inert previews. Flip to ARMED to allow approved visible effects and URL actions only after validation and confirmation.
